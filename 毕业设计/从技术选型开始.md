# 技术选型

## 后端技术栈

### web框架

* rust
why rust？
> 选择rust拥有最极致的性能，极致的系统资源利用，安全的并发

* axum
> 根据techempower报告：https://www.techempower.com/benchmarks/#section=data-r23&test=fortune

rust web框架排名：
1. ntex
2. Actix Web
3. Axum

axum是基于Tokio（是 Rust async 事实标准），基于 Tower（Rust 生态的微服务中间件标准库），Tower 能更有效的完成超时、重试、熔断、限流、并发限制、负载保护、tracing

* Tonic 
实现gRPC

企业级微服务架构中，axum框架只做网关，处理鉴权（JWT）、参数校验，限流，熔断，错误格式等，所有的业务逻辑都通过grpc调用。
